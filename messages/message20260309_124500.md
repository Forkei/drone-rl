# Settling Fix — Failed + Root Cause + Decision Needed

## What happened

Tried the reward changes twice (v2: vel_penalty at 0.3m; v2b: vel_penalty inside warm_zone_thresh).
Both failed immediately after resuming from best_model.

Diagnostic: `clip_fraction = 0.59` persistently across 300k+ steps (normal is 0.05-0.15).
This is 6× normal, meaning PPO is clipping virtually every gradient step.

**Root cause: value function mismatch on reward-change resume.**
PPO.load() restores the full checkpoint including the value head. The value head was
trained on old rewards (warm_zone_bonus=0.5, no vel_penalty). New rewards produce
different returns G', so advantage estimates A = G' - V(s) are wildly off. Every policy
update is immediately clipped. The policy degrades rather than adapts.

Eval reward at best in 280k steps: 1.28 (v1 had 12.58 at peak). Policy collapsed.

## Options

**Option A: Retrain from scratch with v2 reward**
- ~2M steps, ~50 min
- Avoids the value function mismatch (builds V from scratch on new reward)
- Settling fix is baked in from stage 1, so policy learns approach+settle together
- Risk: curriculum must run to completion again

**Option B: Skip settling fix, go straight to DR from v1 best_model**
- No reward changes, avoids collapse
- DR (wind + mass variation) will IMPLICITLY require settling behavior:
  if wind kicks drone away from goal, policy must return AND stay
  → forces the stopping behavior the settling reward was trying to teach
- Faster path to real DR training data
- Settling issues can be reassessed post-DR on the 9 test cases
