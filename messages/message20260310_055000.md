## Technical Discoveries This Session

### SAC ent_coef attribute name
In SB3 v2.7.1, `model.ent_coef_tensor` does NOT exist. Use instead:
```python
float(torch.exp(model.log_ent_coef.detach()))
```
Bug caused first SAC run to crash at step 99k. Fixed in train_nav_sac.py.

### SAC log_ent_coef vs ent_coef_tensor
The first SAC run (using logs_nav_sac/) showed ent_coef=NaN in TensorBoard because
the callback was trying to read the wrong attribute. Fixed run uses logs_nav_sac2/.

### Eval rewards ≠ benchmark score (confirmed again)
Stage4 PPO had negative eval rewards throughout but scored 5/9 on benchmark.
SAC had 24-26% stochastic training success rate but 8/9 on deterministic benchmark.
The EvalCallback uses random positions within target_range. Benchmark uses fixed cases.
The two metrics measure fundamentally different things.

### PPO stage4 training behavior
Despite lr=3e-5 keeping clip_fraction stable (0.11-0.15), the PPO training reward
stayed near zero throughout 2M steps. The policy learned anyway, proving that
the benchmark measures generalization not training curve shape.
